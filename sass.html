Syntactically Awesome Stylesheets

What is Sass?

  • "CSS with superpowers"
  • A stylesheet language that is compiled to CSS
  • Any valid CSS is valid Sass

Installation

Installation is fairly quick and easy.

For Windows (Using Chocolatey Package Manager):

choco install sass

For Mac OS X or Linux (Homebrew):

brew install sass/sass/sass

Compiling

Simply write it, compile it, and pop the CSS file into your web application. 

sass input.scss output.css

You can tell Sass to auto-update (recompile) your output CSS file whenever saving changes to the source Sass file. 

sass --watch input.scss output.css

Sass Variables

Not to be confused with CSS variables (or CSS custom properties)

Like CSS, Sass features the usage of variables that store various kinds of data.

  • Imperative (as opposed to CSS variables being declarative)
  • All compiled away when going from Sass to CSS
  • Utilizes different syntax than CSS variables: $variable-name

Sass 
// Initialize a CSS variable
:root { 
    --font: 'Lato', sans-serif;
}

// Initialize a Sass variable
$arial: 'Arial', sans-serif;

p {
    font-family: $arial;
}

h1 {
    font-family: var(--font);
}
CSS 
:root {
    --font: "Lato", sans-serif;
}

p {
    font-family: "Arial", sans-serif;
}

h1 {
    font-family: var(--font);
}

Interpolation

Interpolation (aka variable substitution) is used to replace an expression containing at least one variable with its value when evaluated.

In Sass, it is denoted by #{$variable-name}

Sass 
.box {
    $width: 40px;
    width: $width;
    height: #{$width * 2};
}
CSS 
.box {
    width: 40px;
    height: 80px;
}

Nesting

Sass supports selector hierarchy. This means that you can style classes when they fall under certain, other parent classes.

You can also nest CSS properties that have the same prefix (eg. font-family & font-weight).

Sass 
article[role="main"] {
    width: 100%;
    padding: 50px;
        
    h1 {
        font: {
            family: 'Times New Roman', serif;
            weight: bold;
        }
        color: red;
    }

    image {
        border: {
            radius: 30px;
            width: 1px;
            style: solid;
        }
    }
}
CSS 
article[role=main] {
  width: 100%;
  padding: 50px;
}
article[role=main] h1 {
  font-family: "Times New Roman", serif;
  font-weight: bold;
  color: red;
}
article[role=main] image {
  border-radius: 30px;
  border-width: 1px;
  border-style: solid;
}

Mixins & Functions

Mixins are essentially macros; they can take arguments and output CSS stylings based off of them. Functions behave as they do in other languages; they can take arguments and then return a specific value.

  • Default values can be specified.
  • Supports flow control (eg. conditionals and iterators)
  • Can support an infinite number of arguments by appending ellipses to the last argument. These arguments are then passed as a list called an argument list
  • Mixins
    • Created using @mixin mixin-name
    • Included into the current styling using @include mixin-name
  • Functions
    • Created using @function function-name(arg1, arg2, arglist...)
    • Called using function-name(x, y, z)

Sass 
// Mixins

@mixin center {
    position: absolute;
    top: 50%;
    left: 50%;
    transform: translate3d(-50%, -50%, 0)
}

@mixin transition($x...) {
    // Ensures cross-browser support
    -webkit-transition: $x;
    -moz-transition: $x;
    -ms-transition: $x;
    -o-transition: $x;
    transition: $x;
}

@mixin fill-bg($image, $pos: center) {
    background: $image;
    background-position: $pos;
}

.box {
    @include center;
    width: 50px;
    height: 50px;
    background-color: black;

    &:hover {
        background-color: red;
        width: 100px;
    }

    @include transition(background-color 0.2s 
        ease-out, width 0.5s ease-out)
}

.image-container {
    @include fill-bg('duckie.png');
}
CSS 
.box {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate3d(-50%, -50%, 0);
  width: 50px;
  height: 50px;
  background-color: black;
  -webkit-transition: background-color 0.2s ease-out, width 0.5s ease-out;
  -moz-transition: background-color 0.2s ease-out, width 0.5s ease-out;
  -ms-transition: background-color 0.2s ease-out, width 0.5s ease-out;
  -o-transition: background-color 0.2s ease-out, width 0.5s ease-out;
  transition: background-color 0.2s ease-out, width 0.5s ease-out;
}
.box:hover {
  background-color: red;
  width: 100px;
}

.image-container {
    background: "duckie.png";
    background-position: center;
}
Sass 
// Functions
@use 'sass:math';
@function average($nums...) {
    $sum: 0;
    @if (length($nums) == 0) {
        @return 0;
    }
    @else {
        @each $number in $nums {
            $sum: $sum + $number;
        }
        @return math.div($sum, length($nums));
    }
}

.circle {
    $diameter: average(20px, 30px, 50px);
    width: $diameter;
    height: $diameter;
    border-radius: 50%;
}
CSS 
.circle {
  width: 33.3333333333px;
  height: 33.3333333333px;
  border-radius: 50%;
}

Modules & Partials

Sass also supports importing external Sass files using the @use 'module'. This means that you can inherit and use Sass members (variables, functions, & mixins) as well as CSS styles.

  • Importing modules and partials adds a namespace to reference Sass members
  • If you want to privatize certain members, you can prepend the member name with a - or _, and they will not be imported into other stylesheets.
  • If you don't want to compile the imported stylesheet, you can use a Sass partial
    • Partials are denoted by a leading underscore in the filename

Sass | module.scss 
@mixin fill-container {
    width: 100%;
    height: 100%;
}

body {
    background-color: black;
    margin: 0;
    padding: 0;
    @include fill-container;
}

* {
    box-sizing: border-box;
}
Sass | index.scss 
@use 'module';

.layout-container {
    @include module.fill-container;
    padding: 100px;
}
CSS | output.css 
body {
  background-color: black;
  margin: 0;
  padding: 0;
  width: 100%;
  height: 100%;
}

* {
  box-sizing: border-box;
}

.layout-container {
  width: 100%;
  height: 100%;
  padding: 100px;
}

Debugging

Sass also offers a neat collection of debugging tools to aid in troubleshooting during compilation

  • @error <expression> Returns an error & prints a stack trace if the expression is true
  • @warn <expression> Prints the value of the expression & a stack trace of how the mixin / function was called
  • @debug <expression> Prints the value of the expression, the filename, & line number

Sass | fib.scss 
@function fib($n) {
    $num: fib-helper($n, 0, 1);
    @debug $num;
    @return $num;
}

@function fib-helper($n, $fib1, $fib2) {
    @if ($n == 0) {
        @return $fib2;
    }
    @return fib-helper($n - 1, $fib2, $fib1 + $fib2);
}

.box {
    width: #{fib(9)}px;
}
shell 
fib.scss:3 Debug: 55
